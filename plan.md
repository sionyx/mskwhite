# План разработки Telegram-бота для продажи доступа Outline через Telegram Stars

## Цель
Собрать Telegram-бота на Python, который проводит пользователя по простому сценарию покупки за 10 Telegram Stars, после успешной оплаты создает доступ в Outline VPN и отправляет пользователю ключ вместе со ссылкой на клиент Outline.

## Базовые технологические решения
- Язык: Python
- Telegram Bot API: библиотека `python-telegram-bot`
- Работа с Telegram Stars и инвойсами: средства платежей Telegram через `python-telegram-bot`
- Работа с Outline: библиотека `outline-vpn-api` или аналогичная Python-библиотека для работы с Outline Server API
- Конфигурация: переменные окружения для токена бота, payment provider token, URL и сертификата Outline API

## Общая стратегия поставки
План разбит на этапы, где каждый следующий этап добавляет один законченный кусок пользовательского сценария. Это позволяет поручать реализацию отдельным ИИ-агентам без сильной связности между задачами.

Дополнительное требование ко всем этапам: каждый этап должен завершаться состоянием проекта, в котором бот можно локально запустить и проверить. На каждом этапе ИИ-агент обязан обновлять шаблон переменных окружения так, чтобы новый исполнитель мог сразу понять, какие значения нужны для запуска именно на текущем шаге.

## Обязательные требования ко всем ИИ-агентам
- после завершения этапа проект должен запускаться локально
- на каждом этапе должен быть актуализирован файл [` .env.example `](.env.example)
- на каждом этапе должна быть краткая инструкция по запуску и проверке
- нельзя оставлять этап в состоянии, требующем ручной доработки для базового запуска
- изменения должны быть ограничены рамками текущего этапа

## Последовательность передачи задач ИИ-агентам
1. Сначала отдать в реализацию этап 1.
2. После приемки этапа 1 передать этап 2.
3. После проверки инвойса передать этап 3.
4. После подтверждения оплаты передать этап 4.
5. После готовности интеграции с Outline передать этап 5.
6. В конце передать этап 6 для упаковки проекта.

## Почему такой план подходит для ИИ-агентов
- каждый этап имеет четкую границу ответственности
- каждый этап добавляет один завершенный кусок функциональности
- каждый этап можно отдельно запустить и проверить
- промпты можно использовать почти без редактирования
- снижается риск, что агент попытается реализовать весь проект сразу
- проще проверять результат после каждого шага

## Примечание по дальнейшему развитию
Вне рамок текущего плана остались хранение выданных ключей, защита от повторной выдачи, журналирование, админские функции, деплой и мониторинг. Их можно вынести в отдельный следующий план после завершения базового сценария.
