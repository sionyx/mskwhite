# Инструкция по запуску Telegram-бота

## Предварительные требования
- Python 3.8+
- Аккаунт Telegram
- Бот, созданный через @BotFather
- Платежный провайдер, настроенный через @BotFather
- Доступ к Outline Server API
- Локальное виртуальное окружение `venv`

## Установка зависимостей
```bash
source venv/bin/activate && pip install -r requirements.txt
```

## Настройка переменных окружения
1. Создайте файл `.env` на основе шаблона:
```bash
cp .env.example .env
```

2. Откройте файл `.env` и заполните переменные:
   - `BOT_TOKEN` - токен вашего бота от @BotFather (production)
   - `PAYMENT_PROVIDER_TOKEN` - токен платежного провайдера от @BotFather (production)
   - `TEST_BOT_TOKEN` - токен тестового бота от @BotFather (опционально)
   - `TEST_PAYMENT_PROVIDER_TOKEN` - токен тестового платежного провайдера (опционально)
   - `OUTLINE_API_URL` - management API URL вашего Outline-сервера
   - `OUTLINE_CERT_SHA256` - SHA-256 fingerprint сертификата Outline API

## Получение токенов
### Токен бота
1. Найдите @BotFather в Telegram
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Скопируйте полученный токен в файл `.env`

### Токен платежного провайдера
1. В @BotFather выберите вашего бота
2. Отправьте команду `/mybots`
3. Выберите "Bot Settings" → "Payments"
4. Настройте платежного провайдера и получите токен
5. Скопируйте токен в файл `.env`

### Параметры Outline API
1. Откройте панель управления Outline Manager
2. Скопируйте management API URL сервера
3. Скопируйте SHA-256 fingerprint сертификата
4. Перенесите оба значения в `.env`

## Запуск бота

### Production режим (по умолчанию)
```bash
source venv/bin/activate && python bot.py
```

### Тестовый режим
```bash
source venv/bin/activate && python bot.py --test
```

**Примечание:** В тестовом режиме бот использует тестовые токены (`TEST_BOT_TOKEN` и `TEST_PAYMENT_PROVIDER_TOKEN`) и подключается к тестовым серверам Telegram.

## Проверка работы
### Проверка команды `/start`
1. Найдите вашего бота в Telegram по имени
2. Отправьте команду `/start`
3. Убедитесь, что бот отвечает приветственным сообщением
4. Проверьте, что кнопка "🛒 Купить доступ" отображается корректно

### Проверка отправки инвойса
1. Нажмите кнопку "🛒 Купить доступ"
2. Убедитесь, что бот отправляет инвойс на 10 Telegram Stars
3. Проверьте, что в инвойсе указано:
   - Название: "VPN-доступ через Outline"
   - Описание: "Доступ к VPN-серверу Outline на 30 дней"
   - Сумма: 10 Telegram Stars

### Проверка интеграции с Outline после успешной оплаты
1. Запустите бота в тестовом режиме командой `source venv/bin/activate && python bot.py --test`
2. Откройте тестового бота в Telegram и выполните сценарий `/start` → "🛒 Купить доступ"
3. Выполните тестовую оплату Telegram Stars
4. Убедитесь, что бот корректно подтверждает pre-checkout запрос
5. Убедитесь, что после успешной оплаты бот:
   - отправляет сообщение о начале создания ключа Outline
   - вызывает Outline API
   - создает новый access key с именем на основе Telegram username или имени пользователя
   - отправляет пользователю строку `Access key: ...`

## Структура проекта
- `bot.py` - основной файл бота
- `outline_service.py` - сервис интеграции с Outline API
- `requirements.txt` - зависимости проекта
- `.env.example` - шаблон переменных окружения
- `.env` - файл с реальными переменными окружения (создается пользователем)
